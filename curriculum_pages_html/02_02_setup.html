Course setup

Step 1. Clone the course repository at github.com/PrefectHQ/write-workflows-course by running this command in your CLI: 

git clone https://github.com/PrefectHQ/write-workflows-course.git

The contents of the repository will be downloaded to your local machine in the write-workflows-course folder.

Step 2. Create and activate a virtual environment.

Virtual environments isolate the Python library versions you need for your projects to help you avoid dependency conflicts.

To create a Python virtual environment for this course. We recommend using uv, but feel free to use another virtual environment manager if you prefer.

Install uv

If needed, install uv on macOS or Linux by running this command in your CLI:

curl -LsSf https://astral.sh/uv/install.sh | sh

If needed, install uv on Windows by running this command in your CLI:

powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

See this page for more information on installing uv.

The `uv.lock` lockfile contains the dependencies for this course. To create a virtual environment and install the dependencies, run this command from the root of the repository run:

uv sync

Activate the virtual environment by running:

source .venv/bin/activate

Alternatively, if you're using venv or conda, you can install the dependencies by running:

pip install -r requirements.txt

Step 3. Connect to Prefect Cloud

You can interact with a Prefect API server one of two ways: either host a server yourself or use Prefect Cloud platform.

With the Prefect Cloud platform, Prefect handles scaling the database and provides features enterprises need, such as:

  • authentication and access controls
  • audit logging
  • hosted compute
  • and additional features to take action based the global state of your workflows.

Throughout this course we'll use Prefect Cloud as our API server.

If you don't already have a Prefect Cloud account, create one at app.prefect.cloud.

If you were invited to a Prefect Cloud account by your organization, follow the instructions in the invitation email to sign in to your account.

Save your API key to your Prefect profile.

From your CLI, authenticate your CLI with :

prefect cloud loginÂ 

This command saves both your API URL and an API key to your local Prefect profile, a file for storing configuration.

Your profile stays active until you switch to a different profile.

Each Prefect Cloud account could have access to multiple workspaces. If you have access to multiple workspaces, choose the workspace you want to use for the course when prompted. Each workspace is completely self contained.

Check your Prefect setup by running

prefect version
