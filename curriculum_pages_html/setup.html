Course setup

Step 1. Clone the course repository at github.com/PrefectHQ/write-workflows-course by running this command in your CLI: 

git clone https://github.com/PrefectHQ/write-workflows-course.git

The contents of the repository will be downloaded to your local machine in the write-workflows-course folder.

Step 2. Create and activate a virtual environment.

Virtual environments isolate the Python library versions you need for your projects to help you avoid dependency conflicts.

To create a Python virtual environment for this course. We recommend using uv, but feel free to use another virtual environment manager if you prefer.

Install uv

If needed, install uv by running this command in your CLI: 

pip install uv

See [this GitHub gist](https://gist.github.com/discdiver/0bb3bf96f02c182f96d45278f9564551) for instructions to install uv

Create a virtual environment by running this command in your CLI:

uv venv

Activate the virtual environment by running this command in your CLI:

uv venv --activate

Step 3. Install the necessary Python packages into your virtual environment by running this command in your CLI:

uv

You should see several packages installed into your environment.

The packages versions pinned here work together as of this writing. If you like, you can remove the pins to upgrade to the latest package versions.

Step 5. Set your Prefect backend option.

You can interact with a Prefect API server one of two ways. You can either host a server yourself or you can interact with a server hosted on the Prefect Cloud platform.

With the Prefect Cloud platform, we take care of scaling the database, provides user authentication and access controls, and additional features for enterprises to understand the global state of their workflows.

Throughout the course we'll demonstrate with a self-hosted Prefect server instance, but either option is fine.

If you are self-hosting a Prefect server instance, the default database type is SQLite. PostgreSQL is a good choice for production workloads, but either database is fine for this course.

Save your API endpoint to your Prefect profile. 

If you're using Prefect server, run

prefect config set PREFECT_API_URL="http://127.0.0.1:4200/api"

This command will save your API endpoint to your active profile. Learn more about saving configuration with profiles in the docs.

If you're using Prefect Cloud instead of self-hosting a server, sign up for a Prefect Cloud account or sign in and run

prefect cloud loginÂ 

to authenticate your CLI. This command saves both your API URL and an API key.

Check your Prefect setup by running

prefect version
